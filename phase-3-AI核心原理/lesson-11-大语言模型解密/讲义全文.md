# 第11课 · 大语言模型解密：GPT到底在做什么

## 一、让计算机理解语言

计算机不认识"猫"这个字。它只知道0和1。所以让AI理解语言的第一步是：**把文字变成数字**。

### 词袋模型（早期做法）
把所有单词当"特征"。一句话里有哪些词，词频是多少。

问题：失去了"词序"和"语义"。"猫在狗前面"和"狗在猫前面"在词袋模型里是一样的。

### 词向量（现代做法）
每个词对应一个高维空间中的向量（一系列数字）。

神奇的是：语义相近的词，在向量空间里距离也近。

```
"猫" → [0.12, 0.53, -0.28, ...]
"狗" → [0.10, 0.51, -0.25, ...]  （和猫距离近）
"手机"→ [0.85, -0.12, 0.33, ...]（和猫距离远）
```

经典的类比：**"国王" - "男人" + "女人" ≈ "女王"**
这说明模型学到的不只是单词，还有概念之间的关系。

## 二、Transformer与注意力机制

### 自注意力
想象你在读这句话：

"小明今天没来上学，因为他生病了。"

当读到"他"时，你自然知道指的是"小明"。Transformer的自注意力机制就是做这件事——**计算句子中每个词和其他词的关系强度**。

### 多头注意力
不是"从一个角度"看关系，而是从多个角度同时看：
- 角度1：语法关系（主语、宾语）
- 角度2：语义关系（谁做了什么）
- 角度3：距离关系（词和词之间多近）

这就像几个人同时看同一张照片，一个人注意构图，一个人注意颜色，一个人注意表情。

## 三、预训练

GPT等大模型通过"下一个词预测"来学习：

**任务：** 给你前面一段话，猜下一个词是什么。

```
输入：今天天气真______
目标：好（或其他合理答案）
```

看起来简单，但在几千亿词的训练数据中，模型学会了：
- 语法规则
- 事实知识
- 推理能力
- 翻译、编程、写作……

### 涌现能力
小模型（几百万参数）只会简单的模式匹配。当模型规模达到某个临界点（约100亿参数以上），**突然出现了小模型没有的能力**，比如思维链推理。这就叫"涌现"。

## 四、Token与上下文

Token是大模型的基本处理单位：
- 1个token ≈ 0.75个英文词
- 1个token ≈ 1个中文字

上下文窗口是模型一次能"记住"的最大token数：
- GPT-4：128k tokens
- Claude：200k tokens
- Gemini：1M tokens

## 五、解码策略

模型生成下一个词时，不是只选"最可能"的那个词：

**温度（Temperature）：**
- 温度=0：每次都选最可能的（稳定但无趣）
- 温度=0.7：有点变化（写作推荐）
- 温度=1.0：变化更大（创意写作）

**Top-p采样：**
只考虑累积概率达到p的这部分词，忽略尾巴上的概率。
