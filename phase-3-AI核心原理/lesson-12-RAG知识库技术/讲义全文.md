# 第12课 · RAG技术：让AI拥有你的专属知识库

## 一、纯LLM的局限

即使是最强的大模型，也有三个致命局限：

**1. 知识截止日期**
ChatGPT的知识截止于某个时间点。问它2025年的事，如果没联网搜索，它不知道。

**2. 幻觉**
AI会"一本正经胡说八道"。它不知道某个答案时，不会说"我不知道"，而是会编造一个看起来合理的答案。

**3. 不知道你的私密数据**
GPT不知道你公司的内部文档，不知道你个人笔记的内容。这些数据也不可能上传去训练大模型。

**RAG就是解决这三个问题的方案。**

## 二、RAG的工作原理

RAG的灵感来自"开卷考试"：
- 纯LLM = 闭卷考试（全靠脑子记）
- RAG = 开卷考试（给了资料，边查边答）

**流程：**
1. 你问一个问题
2. 系统先去"知识库"里找相关的资料
3. 把资料和你问题一起给AI
4. AI基于资料回答

## 三、核心概念：向量嵌入

Embedding就是把"词语/句子/文档"转换成一串数字（向量）。

**关键特性：语义相近 → 向量相近**

"我今天心情很好"和"我今天很开心"的向量距离很近。
"我今天心情很好"和"今天天气很热"的向量距离较远。

这就实现了**语义搜索**——不是匹配关键词，而是理解意思。

## 四、实战工具

### AnythingLLM
最适合个人用户的RAG工具：
1. 下载安装AnythingLLM
2. 选择LLM（可以用OpenAI/Claude/本地模型）
3. 导入文档（PDF/Word/TXT）
4. 直接提问，AI基于文档回答

### Dify
适合团队和专业用户：工作流配置、API调用、权限管理。

## 五、RAG的最佳应用场景
- 企业内部知识库问答
- 个人笔记/文档检索
- 客服系统
- 法律/医疗文档辅助
- 产品说明书问答
